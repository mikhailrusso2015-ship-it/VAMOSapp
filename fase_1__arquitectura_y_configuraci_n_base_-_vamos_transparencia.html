# Estructura del Proyecto: Vamos (Carpooling PWA) ## 1. Árbol de Directorios (Folder Structure) Esta estructura sigue las mejores prácticas de Next.js (App Router) y está diseñada para la modularidad y escalabilidad. ```text vamos-app/ ├── public/ │ ├── icons/ # Iconos de la PWA (manifest icons) │ ├── images/ # Activos estáticos │ └── sw.js # Service Worker personalizado para offline ├── src/ │ ├── app/ # Next.js App Router │ │ ├── api/ # Endpoints de API (Auth, SOS, Matching) │ │ ├── layout.tsx # Layout raíz con Providers │ │ ├── page.tsx # Landing / Home │ │ └── (dashboard)/ # Rutas protegidas (Conductores/Pasajeros) │ ├── components/ │ │ ├── common/ # Botones, inputs neumórficos, modales │ │ ├── forms/ # Formularios de KYC y Registro │ │ ├── layout/ # Navbar, Sidebar, Mobile Nav │ │ └── maps/ # Componentes de Google Maps │ ├── config/ # Configuraciones de Firebase, Cloudinary, etc. │ ├── hooks/ # Custom hooks (useAuth, useMap, useOffline) │ ├── lib/ # Utilidades de terceros (Twilio, Cloudinary) │ ├── services/ # Lógica de negocio (Firestore, Matching Logic) │ ├── store/ # Gestión de estado (Zustand o Context) │ ├── styles/ # Global CSS y utilidades de Tailwind │ └── types/ # Definiciones de TypeScript ├── .env.example ├── .gitignore ├── next.config.js ├── package.json ├── postcss.config.js ├── tailwind.config.js └── tsconfig.json ``` ## 2.
